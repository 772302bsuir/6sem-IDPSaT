1) Консолидация данных - это..
Объединение данных из одной или нескольких областей данных и вывод их в виде таблицы в итоговом листе.
2) Способы консолидации данных:
Консолидация по расположению (данные из различных источников упорядочены одинаково и в них одни и те же подписи)
Консолидация по категории ( данные из различных источников упорядочены по-разному, но в них одни и те же подписи столбцов и строк)
3) Для чего используется ETL?
Для приведения всех данных к единой системе значений и детализации, а также для обеспечения аудиторского следа при преобразовании данных, чтобы преобразования можно было понять
4) Для чего он используется Таленд?
Таленд предоставляет решения для интеграции данных, качества данных, управления данными и подготовки данных.
5) Какие компоненты и разъемы для интеграции данных использовались в работе и для чего?
tFileInputDelimited - Считывает файл за строкой с разделителями, разделяет их на отдельные поля и передает их следующему компоненту.
tMap - Он берет данные из одного или нескольких источников, преобразует их, а затем отправляет преобразованные данные одному или нескольким получателям.
tLogRow -  Он отображает данные / вывод в консоли запуска
